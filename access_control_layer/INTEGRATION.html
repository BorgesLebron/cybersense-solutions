CyberSense.Solutions — Auth Integration Guide

How to wire auth.js and auth.css into every page of the CyberSense frontend.

1. Script load order (all pages)

Add these two lines to the <head> of every HTML page, after styles.css:

<link rel="stylesheet" href="styles.css">
<link rel="stylesheet" href="auth.css">   <!-- ADD THIS -->

At the bottom of <body>, load scripts in this order:

<!-- auth.js MUST load before main.js -->
<script src="auth.js"></script>
<script src="main.js"></script>
auth.js runs initSession() on DOMContentLoaded, which silently refreshes the token and updates the nav and tier gates before the page is interactive.

2. Set the API base URL

Before the script tags, set the API base URL for your environment:

<!-- Production -->
<script>window.CS_API_BASE = 'https://api.cybersense.solutions';</script>

<!-- Development -->
<script>window.CS_API_BASE = 'http://localhost:3000';</script>

3. Page-by-page changes

All pages Login button → auth modal

Replace the static onclick="window.location='dashboard.html'" with:

<!-- BEFORE -->
<button class="nav-login" onclick="window.location='dashboard.html'">Login</button>

<!-- AFTER -->
<button class="nav-login" onclick="CS.Auth.showLoginModal()">Login</button>

auth.js replaces this button with the user menu automatically after login.

index.html Hero CTAs

The "Start Free" button should trigger the registration tab:

<a href="#" class="btn btn-primary btn-lg" onclick="CS.Auth.showLoginModal({tab:'register'});return false;">Start Free</a>

index.html / radar.html / intelligence.html Gate overlays

Add data-subscribe attributes to subscribe CTA buttons. auth.js wires the checkout flow:

<!-- Gate overlay subscribe buttons -->
<a href="#" class="btn btn-primary btn-sm" data-subscribe="monthly">Subscribe</a>

<!-- Free signup -->
<a href="#" class="btn btn-primary" data-subscribe="freemium">Subscribe Free</a>

radar.html / intelligence.html Blurred content with tier

Add data-min-tier attributes to blurred content so auth.js can unlock it after login:

<div class="article-card blurred" data-min-tier="monthly" aria-hidden="true">
  ...article content...
</div>

newsletter.html Freemium banner

Add the freemium banner div — auth.js hides it automatically for freemium+ users:

<div class="freemium-banner" style="display:none">
  <p>You're reading as a guest. Subscribe free for full access.</p>
  <button class="btn btn-primary btn-sm" onclick="CS.Auth.showLoginModal({tab:'register'})">Subscribe Free</button>
</div>

dashboard.html Admin auth

The dashboard login form is handled by auth.js automatically. No changes needed to the form HTML — auth.js detects #login-form + #login-gate + #dashboard-shell and wires the real API call.

The nav "Sign Out" button is already wired in dashboard.html. Confirm the ID matches:

<button class="nav-login" id="nav-logout" style="display:none;" onclick="CS.Auth.logout()">Sign Out</button>

For the dashboard metric elements, add IDs so auth.js can populate them with live API data:

<!-- Metric cards in dashboard-shell -->
<div class="metric-val" id="metric-subscribers">—</div>
<div class="metric-val" id="metric-paid">—</div>
<div class="metric-val" id="metric-enterprise">—</div>
<div class="metric-val" id="metric-mrr">—</div>
<div class="metric-val" id="metric-risks">—</div>

<!-- Activity feed container -->
<div class="activity-feed" id="activity-feed"><!-- populated by auth.js --></div>

<!-- Pipeline status bars -->
<div id="pipeline-status-bars"></div>

<!-- Agent health -->
<div id="agent-health-summary"></div>

zerotrust.html Pricing CTAs

<!-- Monthly plan -->
<button class="btn btn-primary" data-subscribe="monthly">Get Monthly Access</button>

<!-- Enterprise plan -->
<button class="btn btn-primary" data-subscribe="enterprise">Contact Sales</button>

training.html Locked buttons

Locked training buttons are automatically unlocked by auth.js for monthly+ subscribers. The HTML already has the correct btn-locked class. No change needed.

4. Account page (account.html) — new page needed

Create account.html with a bare container — auth.js renders the account panel:

<section class="section">
  <div class="container">
    <div id="account-panel">
      <!-- auth.js renders account UI here after login -->
      <p style="color:#94a3b8">Loading your account...</p>
    </div>
  </div>
</section>

5. Email verification (query param)

After registration, the verification email links to:

https://cybersense.solutions/index.html?token=VERIFY_TOKEN

auth.js detects the token param on page load, calls the verify endpoint, and shows a success toast. No extra page needed.

6. Password reset (query param)

The reset email links to:

https://cybersense.solutions/index.html?reset=RESET_TOKEN

auth.js detects the reset param and injects the reset password form into the page automatically.

7. Public API surface

All auth functions are available globally via CS.Auth:

MethodDescription
CS.Auth.showLoginModal(options)Opens the auth modal. Options: tab, message, onSuccess, redirect
CS.Auth.closeModal()Closes the auth modal
CS.Auth.logout()Logs out, clears tokens, updates UI
CS.Auth.isLoggedIn()Returns boolean
CS.Auth.getTier()Returns 'free' | 'freemium' | 'monthly' | 'enterprise'
CS.Auth.hasMinTier(tier)Returns boolean — checks if user meets minimum tier
CS.Auth.getUser()Returns user object from sessionStorage
CS.Auth.startSubscription(tier)Initiates Stripe checkout. Prompts login if not authenticated.
CS.Auth.manageBilling()Opens Stripe billing portal
CS.Auth.apiFetch(path, options)Authenticated fetch wrapper with auto-refresh
CS.Auth.showToast(message, type)Shows toast. Types: success, error, warning, info

8. Custom events

Listen for auth state changes on document:

document.addEventListener('cs:login',  (e) => console.log('Logged in:', e.detail));
document.addEventListener('cs:logout', ()  => console.log('Logged out'));
document.addEventListener('cs:guest',  ()  => console.log('Guest session'));

9. Environment variables

VariableValue
window.CS_API_BASEAPI server URL. Default: https://api.cybersense.solutions
Never set window.CS_API_BASE to a localhost URL in production builds. Use a build step or environment-specific HTML file to inject the correct value.
