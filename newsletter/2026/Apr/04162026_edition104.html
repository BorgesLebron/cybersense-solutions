Patch Tuesday, 167 Flaws, and a Library Nobody Checked

Microsoft's April 2026 Patch Tuesday lands with 167 vulnerabilities patched and two actively exploited zero-days confirmed. It arrives in the same week that North Korea successfully poisoned the Axios npm package – one of the most downloaded JavaScript libraries in existence with over 100 million weekly installs – in a supply chain attack that reached development environments before detection. More than 100 malicious extensions were discovered in the Chrome Web Store stealing OAuth tokens and deploying backdoors. And Booking.com confirmed a data breach affecting customers across multiple countries.

The pattern is consistent with what this brief has been tracking all week: attackers are operating through trusted infrastructure. The Patch Tuesday zero-days were exploited before patches existed. The Axios poisoning reached developers through a package they had used reliably for years. The Chrome extensions passed marketplace review. Each incident exploits the gap between trust established over time and verification applied in the moment. This edition covers all three threat items, plus the device management platform abuse that wiped 80,000 devices in 79 countries with a single stolen credential.

Situational Awareness

Microsoft April 2026 Patch Tuesday – 167 Vulnerabilities Including 2 Actively Exploited Zero-Days

Microsoft's April 2026 Patch Tuesday addresses 167 security vulnerabilities across Windows, Office, Azure, and related products, including two zero-day vulnerabilities confirmed as actively exploited in the wild prior to patch availability. Both Windows 10 and Windows 11 cumulative updates are available. For organizations operating on standard monthly patch cycles, April's Patch Tuesday represents a significant remediation load – 167 vulnerabilities across a complex product surface requires careful prioritization. The two zero-days should be treated as immediate – meaning tested and deployed within 24 to 48 hours regardless of where they fall in the normal patch queue. For the remaining 165 vulnerabilities, standard CVSS-guided prioritization applies, with particular attention to any remote code execution and elevation of privilege findings affecting internet-facing systems and domain controllers.

North Korea Poisons Axios npm Package – 100 Million Weekly Downloads, Supply Chain Compromise Confirmed

North Korean threat actors successfully compromised the Axios npm package – one of the most widely used JavaScript HTTP client libraries with over 100 million weekly downloads – in a supply chain attack that distributed malicious code through the official npm registry. OpenAI has since responded by rotating its macOS application certificates following confirmation of exposure. The scale of Axios's adoption makes this one of the highest-reach supply chain attacks documented in 2026: any JavaScript or Node.js project that pulled an affected version of Axios during the compromise window should be treated as potentially exposed. Development teams should audit their dependency trees for the affected version range, rotate any credentials that may have been accessible in affected environments, and review recent deployment activity from systems that executed affected builds. The attack reinforces a pattern: North Korean threat actors have demonstrated consistent capability and willingness to target developer infrastructure at scale.

100+ Malicious Chrome Extensions Discovered Stealing OAuth Tokens and Deploying Backdoors

More than 100 malicious extensions in the official Chrome Web Store have been confirmed as actively stealing Google OAuth2 Bearer tokens, deploying backdoors, and conducting ad fraud across affected installations. OAuth Bearer tokens are session credentials – stealing them allows an attacker to authenticate as the victim to any service the token was issued for, without requiring the user's password or bypassing MFA. This is a direct continuation of the browser extension threat surface covered in Edition 103: extensions that pass marketplace review are not necessarily safe, and the permissions model that makes extensions useful is the same model that makes them dangerous when malicious. Organizations that have not yet conducted a browser extension audit should do so now – the ten-minute process outlined in Edition 103's training byte applies directly to this threat.

Training Byte

How to Audit Your npm Dependencies – Five Commands That Surface Supply Chain Risk

Why it matters now:

The Axios poisoning and the North Korea npm supply chain campaign are not isolated incidents – they are part of a documented pattern of targeting developer package ecosystems. If your organization builds or deploys JavaScript or Node.js applications, your dependency tree is an attack surface.

Five commands to run today:

npm audit – scans installed packages against known vulnerability database. Run it in every project directory.

npm audit --audit-level=critical – filters to critical severity only for triage prioritization.

npm list --depth=0 – shows top-level dependencies. Know what you're directly depending on.

npm outdated – shows packages with newer versions available. Outdated packages accumulate unpatched vulnerabilities.

npx better-npm-audit audit – provides more detailed output than the default npm audit for large dependency trees.

You cannot secure what you have not inventoried.

Career Development

GitHub Advisory Database – Free Vulnerability Intelligence for Every Developer

The GitHub Advisory Database is a free, publicly accessible database of security vulnerabilities in open source software, maintained by GitHub's security team and the broader community. It covers vulnerabilities across npm, PyPI, Maven, RubyGems, Go, and other major package ecosystems – with CVE mappings, severity ratings, affected version ranges, and patched version information. For developers and security practitioners who work with open source dependencies, the Advisory Database provides a reliable, searchable reference for understanding whether a specific package version is affected by a known vulnerability. It integrates directly with Dependabot and GitHub's native dependency scanning, but is also freely accessible via the web interface and API for organizations using other tooling. Building the habit of checking the Advisory Database before introducing a new dependency — or when a supply chain incident like the Axios compromise is reported – is a low-cost, high-value security practice.

Modernization and AI Insight

Single Stolen Credential Wipes 80,000 Devices Across 79 Countries via Microsoft Intune Abuse

The Iran-linked Handala group used a single stolen credential to abuse Microsoft Intune – a device management platform– to wipe approximately 80,000 devices across organizations in 79 countries. The attack demonstrates the compounding risk created by device management platforms: tools designed to give IT administrators broad, efficient control over organizational endpoints become high-value targets precisely because of that breadth. A single privileged credential to a device management console can reach every managed device in an organization. For security teams evaluating their device management posture, this incident makes the case for several specific controls: MFA enforcement on all Intune admin accounts without exception, conditional access policies restricting Intune access to known networks and compliant devices, privileged identity management with just-in-time access for administrative functions, and audit logging with alerting on bulk device actions. The Handala group's capability to execute at this scale with a single credential is a direct consequence of insufficient access controls on a high-privilege platform.

Tax Season Phishing Escalation – 100+ Campaigns Documented, RMM Tool Installation Confirmed as Primary Objective

Proofpoint's threat research team documented over 100 distinct tax-themed phishing campaigns active in early 2026, representing a meaningful escalation in volume and variety compared to prior tax seasons. The campaigns divide along two lines: IRS impersonation lures harvesting credentials, and a second category specifically designed to install remote monitoring and management tools – Datto, N-Able, RemotePC, and ScreenConnect – giving attackers persistent footholds on victim machines well beyond the initial interaction. The RMM installation objective is significant because it shifts the campaign goal from a single credential theft event to persistent access maintained indefinitely after the initial compromise. For organizations with employees who handle tax documents, file returns, or manage payroll, this is a current and active threat – not a seasonal one that ends with the April filing deadline. Employee awareness of IRS impersonation lures and unexpected RMM tool installation requests should be reinforced now.

Final Thought

One hundred and sixty-seven vulnerabilities patched in a single Tuesday. Axios – a library used in tens of millions of projects – poisoned through its official distribution channel. Over a hundred malicious extensions that passed marketplace review. Eighty thousand devices wiped with one credential. None of these required a sophisticated nation-state capability gap to exploit. They required patience, access to trusted infrastructure, and the confidence that most organizations would not be looking. The defense is not a more expensive tool. It is consistent verification of what you trust – and the discipline to do it before an incident makes it urgent.
