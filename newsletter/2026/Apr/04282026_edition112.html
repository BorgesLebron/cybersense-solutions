Management Plane Under Siege: Fortinet, Samsung, and AI-MFA Bypass

Three distinct attack surfaces define this threat briefing, each targeting a layer of enterprise infrastructure that organizations manage but rarely scrutinize with the same rigor as primary systems. CVE-2026-35616 in Fortinet's FortiClient EMS (CVSS 9.1, pre-authentication, and under active exploitation since March 31) allows an unauthenticated attacker to execute arbitrary commands on the management console governing VPN profiles and firewall policies for every managed device. Simultaneously, Samsung MagicINFO 9 Server, the digital signage platform running across enterprise campuses and healthcare facilities, landed on CISA's KEV catalog on April 24 with confirmed Mirai botnet exploitation and a federal May 8 deadline. Compounding the threat landscape, Storm-1747’s Tycoon2FA platform has industrialized MFA bypass by intercepting session tokens in real time. Linked to over 100,000 compromised organizations, its AI-enhanced infrastructure now generates more than 62% of the phishing volume blocked by Microsoft monthly.

The structural thread connecting all three is deployment context. FortiClient EMS, MagicINFO, and Tycoon2FA target what security teams often consider "supporting infrastructure" — endpoint managers, signage servers, and authentication flows. Detection coverage is thinnest here, patch cycles are longest, and a compromised management plane can silently reshape the posture of everything it governs before a single alert fires. The question this week is not whether your primary systems are protected; it is whether the tools you use to protect those systems are themselves in scope for your threat model.

Situational Awareness

FortiClient EMS: Critical Pre-Auth RCE Under Active Exploitation

An improper access control flaw (CWE-284) in Fortinet FortiClient EMS versions 7.4.5 and 7.4.6 allows an unauthenticated attacker to send specially crafted API requests that bypass authentication and authorization checks, executing arbitrary OS commands on the management server with elevated privileges. watchTowr's honeypots logged the first exploitation attempts on March 31, 2026 — before a patch existed. CISA added CVE-2026-35616 to the KEV catalog on April 6 with a federal deadline of April 9, the shortest in the April batch for non-Cisco CVEs. FortiClient EMS is the central control plane for Fortinet endpoint deployments: it distributes VPN profiles, manages firewall policy assignments, and enforces endpoint compliance rules across every registered device. A compromised EMS server does not just represent a foothold — it gives an attacker the administrative keys to reshape endpoint posture silently across the entire managed fleet before any detection fires. Fortinet released a hotfix for affected versions; the full fix ships in 7.4.7. Priority actions: apply the hotfix immediately, restrict EMS API access to management VLAN only, and audit EMS access logs from March 31 forward for anomalous API activity.

Samsung MagicINFO: CISA KEV Alert – Mirai Botnets Targeting Campus Servers

CVE-2024-7399 is an improper path restriction vulnerability in Samsung MagicINFO 9 Server versions prior to 21.1050 that allows an unauthenticated attacker to write arbitrary files (including JSP web shells) with SYSTEM-level privileges on the underlying Windows host. CISA added it to the KEV catalog on April 24 with a federal remediation deadline of May 8; active exploitation tied to Mirai botnet variants was confirmed at time of cataloging. MagicINFO manages digital signage content across enterprise campuses, hospital networks, and retail environments — typically deployed on interior Windows servers with broad internal network access and minimal internet-facing hardening review. This class of deployment creates a meaningful pivot point: a compromised MagicINFO server carries SYSTEM-level process execution, internal network adjacency, and almost no behavioral baseline to detect anomalous activity against. Separately, CVE-2025-4632 (CVSS 9.8) (a bypass of the original patch for the same vulnerability class) remains in active exploitation campaigns; organizations that patched CVE-2024-7399 without applying subsequent updates may still be exposed. Patch to MagicINFO 9 Server 21.1050 or later, segment MagicINFO from general internal access, and verify patching covers both CVE chains.

Storm-1747 Tycoon2FA: The AI-Driven Industrialization of MFA Bypass

Microsoft's April 2026 threat intelligence report details the operational scale of Tycoon2FA, a subscription adversary-in-the-middle (AiTM) phishing platform operated by Storm-1747 that has been linked to over 100,000 compromised organizations since 2023. Tycoon2FA defeats MFA not by stealing credentials but by relaying them in real time: a victim authenticates through a transparent proxy that captures the live session token, which the attacker then replays to authenticate as the user without triggering any MFA prompt. At peak activity, the platform was responsible for roughly 62% of all phishing attempts Microsoft was blocking monthly. The April update documents AI integration across the operation: LLM-generated phishing lures with per-target personalization, automated infrastructure provisioning that spins new proxy domains at scale, and real-time triage of captured sessions to prioritize high-value accounts. The key defensive gap Tycoon2FA exploits is the assumption that any successful MFA authentication is a valid user — session token replay bypasses that entirely. Priority actions: deploy Conditional Access policies tied to device compliance and Continuous Access Evaluation (CAE); migrate from session-token-based MFA to phishing-resistant FIDO2 or passkeys where feasible; and audit Entra ID or equivalent sign-in logs for AiTM indicators — impossible travel, unfamiliar device strings, and token replay patterns across short time windows.

Training Byte

Targeting the Manager: Five Defenses for the FortiClient EMS Gap

The vulnerability:

CVE-2026-35616 exploits an access control gap in FortiClient EMS's API layer — an unauthenticated request bypasses all authorization checks and reaches command execution on the server managing endpoint policy for the entire fleet. EMS generates no alert for its own exploitation. The endpoint security policies it distributes — VPN configurations, firewall rules, compliance enforcement — can be silently modified from a single compromised API session.

Five actions while the patch deploys:

  1. Apply the hotfix now. Fortinet released a fix for 7.4.5 and 7.4.6; deploy it before the full 7.4.7 release. No patch means live exposure on every EMS-managed device.
  2. Restrict the EMS API to a management VLAN. CVE-2026-35616 requires network access to the EMS API. A management plane that isn't internet-facing or accessible from general corporate subnets is not reachable.
  3. Audit EMS access logs from March 31 forward. Exploitation was confirmed from that date. Anomalous API calls — especially unauthenticated requests or unexpected policy modifications — are the indicator.
  4. Validate endpoint policy integrity. If EMS was accessible and unpatched, treat managed endpoint configurations as potentially tampered. Re-push known-good policy baselines from a trusted state.
  5. Map what EMS controls. VPN profiles, firewall assignments, and compliance posture checks are all within EMS scope. Understanding the blast radius of an EMS compromise defines what to audit first if exploitation is confirmed.

When the tool distributing your security policies is the exploit surface, the policy itself is the attacker's first target.

Resilient Architecture: Defending a Compromised Management Plane

CVE-2026-35616 and Tycoon2FA share a common architectural failure point: both exploit the implicit trust organizations extend to management-layer systems. FortiClient EMS is trusted to push policy. MFA tokens are trusted to represent authenticated users. When those trust assumptions are wrong, the downstream impact is unbounded. Security Architecture roles — specifically those focused on Zero Trust design — are where this week's failure patterns get addressed structurally.

Zero Trust Architecture (ZTA) roles are up 34% year-over-year (CyberSeek, Q1 2026), with the largest demand from organizations restructuring after management-plane compromises. The skills driving the current premium: identity-centric access design (Conditional Access, continuous authentication, FIDO2 deployment); least-privilege segmentation of management infrastructure (separating management plane access from production plane — exactly the control that limits CVE-2026-35616's blast radius); and MITRE ATT&CK-mapped threat modeling (T1190 for public-facing application exploits, T1556 for MFA bypass). NIST SP 800-207 is the foundational ZTA reference document.

Modernization and AI Insight

Entra ID Alert: Microsoft Details AI-Automated Device Code Phishing

Microsoft's April 6, 2026 security blog details a newly documented campaign class: AI-driven device code phishing operations that automate the full attack lifecycle from target selection through token capture without manual attacker involvement. The campaign targets enterprise Entra ID tenants using legitimate Microsoft OAuth device code flows — sending phishing lures that direct victims to enter a device authorization code at an attacker-controlled page, which the AI-managed infrastructure relays in real time to capture a valid access token. Unlike traditional phishing, there is no credential collection and no password reset to invalidate the session — the captured OAuth token grants persistent access to mail, Teams, SharePoint, and connected SaaS applications until token expiration or explicit revocation. The operational distinction from prior campaigns is the infrastructure: AI handles target persona development, lure generation, relay provisioning, and session triage with no human operator required between initial targeting and token capture. Organizations should restrict device code flow authentication in Conditional Access policies, monitor for OAuth token grants from unregistered or non-compliant devices, and treat abnormal OAuth application consent grants as a high-priority alert condition.

Final Thought

This week's threat briefing shares a single structural observation: the infrastructure organizations use to manage other infrastructure carries an implicit trust that attackers have learned to exploit before defenders think to defend it. FortiClient EMS is trusted to know what policies every endpoint should enforce — until it isn't. MagicINFO is trusted to push content to screens across the environment — until it's pushing web shells. MFA is trusted to confirm identity — until an AiTM relay captures the session token before the user's authentication completes. Each case represents a management layer that was never formally included in the threat model. The useful question to carry into this week is not whether your primary systems are hardened. It is whether every system that manages, configures, or authenticates access to those systems is in scope for the same threat modeling, the same patch discipline, and the same detection coverage you apply to the things they govern. Management-plane compromise is pre-ransomware staging of the highest order. Is it in your model?
