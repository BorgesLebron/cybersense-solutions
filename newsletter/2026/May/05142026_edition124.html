Edition 124 - Injection Risk, Credential Boundaries, and Identity Modernization

May 14, 2026
Executive read: Today's risk pattern is boundary failure. Kestra shows the cost of unsanitized input in automation platforms, OpenStack Ironic shows how credential forwarding can cross trust boundaries during import workflows, and D-Link router exposure shows why edge devices with public proof-of-concept activity need fast containment decisions. The innovation and growth items point to the same operating lesson: identity modernization and cyber workforce maturity only matter when processes are clear enough to execute.

Threat Watch

Critical - CVE-2026-38428 - CVSS 9.8 - Kestra SQL injection

Kestra automation workflows expose SQL injection risk

Kestra v1.3.3 and earlier are vulnerable to SQL injection through user-controlled GET input concatenated into database queries without sufficient sanitization or parameterization.

  • Review exposed Kestra deployments and upgrade based on vendor guidance.
  • Prioritize instances reachable from untrusted networks.
  • Hunt for abnormal query patterns or unexpected workflow metadata access.
High - CVE-2026-42997 - CVSS 7.7 - OpenStack Ironic credential forwarding

OpenStack Ironic import flow can leak authorization to remote endpoints

An Ironic iDRAC import issue can forward a time-limited Keystone token or configured storage credentials to a remote endpoint requested during import.

  • Upgrade to fixed Ironic versions where applicable.
  • Review import workflow access controls and remote endpoint validation.
  • Rotate credentials if suspicious import activity is found.
Critical - CVE-2026-7854 - CVSS 10.0/9.8 - D-Link DI-8100 buffer overflow

D-Link router overflow has public exploit indicators

D-Link DI-8100 firmware 16.07.26A1 is affected by a remotely reachable buffer overflow in the POST parameter handler for /url_rule.asp. Public exploit information has been disclosed.

  • Remove affected devices from internet exposure where possible.
  • Review vendor guidance and replacement options for unsupported hardware.
  • Monitor edge-device traffic for suspicious POST activity.
