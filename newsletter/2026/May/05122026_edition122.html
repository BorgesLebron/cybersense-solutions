Threat Radar

Ivanti Endpoint Manager Mobile remote code execution exposure

Ivanti Endpoint Manager Mobile contains an improper input validation flaw that can allow a remotely authenticated administrator to achieve remote code execution. Because the product sits in the mobile device management layer, affected organizations should treat remediation as an identity, endpoint, and management-plane priority.

Required action: apply vendor mitigations or patches, validate administrative account exposure, and remove unsupported deployments if mitigation is unavailable.

CVE-2026-6973 | Severity: critical | Source: https://nvd.nist.gov/vuln/detail/CVE-2026-6973

OpenCTI privilege escalation threatens intelligence integrity

OpenCTI versions 6.6.0 through 6.9.12 include a privilege escalation vulnerability that can allow unauthenticated attackers to query the API as existing users, including the default administrator account. For security teams that rely on OpenCTI as a system of record, this is not only an access-control issue; it can undermine trust in intelligence workflows.

Required action: upgrade to OpenCTI 6.9.13 or disable the default administrator with external management controls where upgrade timing is constrained.

CVE-2026-27960 | Severity: critical | Source: https://nvd.nist.gov/vuln/detail/CVE-2026-27960

ModSecurity denial-of-service condition affects WAF reliability

Libmodsecurity versions before 3.0.15 can crash worker processes when a rule using the hexDecode transformation inspects a query string parameter containing a single character. While service can resume after the attack stops, repeated triggering can reduce the defensive reliability of web application firewall deployments.

Required action: upgrade libModSecurity3 to 3.0.15 and review custom rules that use transformation logic against untrusted query parameters.

Verifiable digital credential issuance

NIST's review of mobile driver's licenses and verifiable credential issuance shows that credential trust depends on both format and issuance process. Standards activity across FIDO, ISO, and OpenID Foundation is pushing identity proofing toward more interoperable, wallet-based trust models.
